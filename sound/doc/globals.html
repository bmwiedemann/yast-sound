YaST2 - Sound configuration

Data structures

Saving structure

For saving module here is just one global saving structure (map):
Sound saving configuration map
NameTypeDescription
cardslistlist with all card-dependent information
systemmapmap all system dependent information
othertermfunction to call to provide some additional work

Each element of the cards list is a map:
One card saving entry
NameTypeDescription
moduleslistlist of maps for /etc/modules.conf
/etc/modules.conf map
NameTypeDescription
aliasstringAlias name
optionsmapOptions for the module
volumemapmap for storing volume, map keys should be channels
othertermfunction to call to provide some additional work

System entry consists now from these keys:
Sound system dependent saving configuration
NameTypeDescription
moduleslistlist of maps for 'modules.conf'
volumemap"mixer" ... mixer command to set-up the volume (must include $CHANNEL and $VOLUME substrings, i.e: "/usr/bin/amixer set \"$CHANNEL\" $VOLUME% unmute")
rcmapmap with rc.config entries (i.e. START_ALSA, START_ALSA_SEQ entry etc ...)
