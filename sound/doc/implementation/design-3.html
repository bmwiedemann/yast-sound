3. Related system configuration files

Just this two files are accessed directly. Some other files (/etc/asound.conf) are accessed indirectly by calling external programs (alsactl).

3.1 /etc/modules.conf

This file is used for reading/writing. Currently reading is used just for removing previously made entry for the sound (ALSA) modules. Writing add to the end of the file sound card configuration, so the start (/etc/rc.d/alsasound) script can be used to start sound.

Thi file is accessed by anyagent (which is implemented in this module) and reads/writes line by line whole file. Some better agent should be used in the future.

3.2 /proc/asound/cards

This file is used for reading and is parsed just to test the sound card configuration. There is anyagent in the SCR path (``.proc.asound.cards'') that reads this file.
