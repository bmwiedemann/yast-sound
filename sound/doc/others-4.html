4. Testing the instalation

requires: attempt of configuring sound card

provides: answer to ``Is instalation correct?'' (goal)

4.1 Doing cat /dev/sndstat

Reading the sound driver status device file should provide additional information on whether the sound card driver initialized properly. Sample output should look something like this (example from Sound-HOWTO): 

% cat /dev/sndstat
Sound Driver:3.5.4-960630 (Sat Jan 4 23:56:57 EST 1997 root, 
Linux fizzbin 2.2.27 #48 Thu Dec 5 18:24:45 EST 1996 i586) 
Kernel: Linux fizzbin 2.2.27 #48 Thu Dec 5 18:24:45 EST 1996 i586 
Config options: 0
 
Installed drivers: 
Type 1: OPL-2/OPL-3 FM 
Type 2: Sound Blaster 
Type 7: SB MPU-401 

Card config: Sound Blaster at 0x220 irq 5 drq 1,5 
SB MPU-401 at 0x330 irq 5 drq 0 
OPL-2/OPL-3 FM at 0x388 drq 0 

Audio devices: 
0: Sound Blaster 16 (4.13) 

Synth devices: 
0: Yamaha OPL-3

Midi devices: 
0: Sound Blaster 16 

Timers: 
0: System clock 

Mixers: 
0: Sound Blaster

The command above can report some error messages. ``No such file or directory'' indicates that you need to create the device files. ``No such device'' means that sound driver is not loaded.

requires: nothing

provides: info of the sound card driver initialization

4.2 Playing a sound

This is possibly the best way for testing all devices. The test differs on kind of device to test.

The simplest way to test audio device can be done by typing e.g. cat sound.au > /dev/audio.

All other devices needs special utility for playing a sound (e.g. playmidi for playing sound through midi device, etc.).

requires: utility for testing device

provides: producing a sound (or noise)

4.3 lsmod output

The status of the modules and which modules depend on which other modules may be checked by /sbin/lsmod. The output should show something like the following (OSS/Free modules): 

sb                     26280   0
uart401                 5640   0  [sb]
sound                  57112   0  [sb uart401]
soundlow                 208   0  [sound]
soundcore               1968   8  [sb sound]

requires: nothing

provides: modules status (are loaded correctly?)
